1.计算机网络概述

internet:通用名词，泛指多个计算机网络连接而成的网络。协议可以是任意的

Internet:泛指全球最大的特定计算机网络

边缘部分:由所有链接在互联网上的主机组成。包括智能摄像头、手机、智能手表

  • 处在Internet边缘的部分就是连接在Internet上的所有主机。这些主机又称为端系统(end system)

  • 主机A和B通信，实际上指的是运行在主机A上的某个程序和运行在主机B上的另一个程序程序进行通信

  • 计算机进程之间的通信简称为计算机之间通信

  • 在网络边缘的end system中运行的程序之间的通信方式可以划分为

1. C/S Clinet/Server Client和Server都是指通信中所涉及的两个应用程序

2. 对等方式(P2P)方式: Peer-to-Peer方式

    - 两个主机在通信时并不区分哪一个是服务请求还是服务提供方

    - 只要两个主机都运行了P2P软件，它们就可以进行平等的、对等连接通信

    - P2P连接方式从本质上看仍然是使用客户服务器方式，只是每一个主机既可以作为服务端也可以作为客户端

    - P2P也会经过服务器，进行管理
    
    在网络核心部分起特殊作用的是**router(路由器)**

三种交换方式：

  1. Circuit Switching电路交换

    • 建立连接(分配通信资源)

    • 通话(一直占用通信资源)

    • 释放连接(归还通信资源)

  1. Packet Switching分组交换


边缘系统连接到核心系统进行通信

  • 采用存储转发技术。把一个message(报文)划分为几个分组后在进行传送，在每个数据段前面加上必要的控制信息组成的header后，构成了一个packet(分组)，又称为包,header可成为包头，用来进行差错校验

  • $packet + header) * n = message$

  1. Message Switching(报文交换): 不分组的进行交换，同样采取存储转发技术

网络结构

  1. 星形结构

  2. 总线结构

  3. 环形结构

  • 这种结构的网络形式主要应用于令牌网中，各设备直接通过电缆串接，最后形成一个闭环，整个网络发送的信息在这个环路中传递，通常称之为”令牌环网“

  • 实际情况，环的两端是通过阻抗匹配器来实现环的封闭，受限于地理位置等环两端的物理连接很难实现

Token在闲状态不断在环网中传递，当有一台主机想要传输数据，它取得令牌，将Token修改为忙状态，并且绕环一圈告知所有环内其他主机，然后获得令牌的主机开始占用信道发送数据，发送完毕后又将Token修改为闲状态以为下一台主机的传输做准备。

  1. 网状型

  • 每一个节点(交换机/路由器)都至少和其他两个节点相连接，具有较高可靠性，但结构复杂，费用较高
  • 可选用最佳路径，传输延迟低

应用最广泛的是星形结构(局域网)和网状型结构。

性能指标

  1. 速率

    单位:bit/s

    计算机发送的信号是以二进制数字形式的，连接在计算机网络上的主机在数字信道上传送比特数据的速率，也成为比特率或数据率

  2. 带宽

    单位:bit/s

    在计算机网络中，带宽用来表示网络中某通道传送数据的能力，因此网络带宽表示在单位时间内网络中的某信道能通过的最高数据率

  3. 吞吐量

    表示在单位时间内通过某个网络(或信道、接口)的数据量

    吞吐率与宽带的区别是吞吐量代表的是实际而带宽代表的是理论最大

  1. 时延(delay或latency)

    数据从网络的一段传送到另一端所需的时间。它有时也成为延迟或迟延

网络中的时延是由以下几个不同的部分组成的：

  • 发送时延 $发送时延=数据帧长度(bit)/发送速率(bit/s)$

在家庭中，边缘设备连接至路由器访问网络，在这种结构中，发送时延的计算应取从主机发送到路由器并由路由器发出的时延作为发送时延，并受最低速率限制

  • 传播时延 $传播时延=信道长度(m)/电磁波在信道上的传播速率(m/s)$

  • 处理时延

处理时延指的是主机或者路由器在收到而分组信息时花费的时间，一般忽略不计，因为没法估量

  1. 往返时间

    RTT(Round-Trip Time)重要指标，表示从发送端将数据发送道传输媒介开始，到发送端收到来自接收端的确认总共经历的时间(不包含发送时延).

    RTT由三部分决定: 链路的传播时间、末端系统的处理时间、路由器的缓存中的排队和处理时间，其中前两个值作为一个TCP连接相对固定，而排队和处理时间会随着整个网路拥塞成都变化而变化，故RTT的变化在一定程度上反映了网络拥塞程度的变化

    $有效数据率=数据长度/(发送时间+RTT(往返时间))$

    由A向B发送数据为例，发送时间为数据长度与自身发送速率的比值，而RTT包括A发送到B途中路由器的排队时延和传播时延以及由B发往A的确认信号达到A的总时间，在到达B后B会有一段处理时间并回复一个确认信号，暂且假定处理时间可以忽略不计，而确认信号的信息长度很短，故B的发送时间也可忽略不计

  2. 利用率

  1. 丢包率

    丢包率即分组丢失率，是指在一定的时间范围内，传输过程中丢失的分组数量与总分组数量的比率

    分组丢失主要有两种情况

    - 分组在传输过程中出现误码，被结点丢掉

- 分组到达一台队列已满的分组交换机时被丢弃，在通信量较大时就可能造成网络拥塞

2.计算机网络体系结构

1.体系结构介绍

开放系统互连参考模型，简称OSI，共七层，从下往上为物理层、数据链路层、网络层、运输层、会话层、表示层、应用层，未流行

TCP/IP体系结构共四层，包括应用层、运输层、网际层和网络接口层

在学习计算机网络原理往往采取这种办法，即综合OSI和TCP/IP的优点，采用一种只有五层协议的体系结构，都包括物理层、数据链路层、网络层、运输层、应用层

·

物理层

导向型：同轴电缆、双绞线、光纤
非导向型：wifi、bluetooth

数据链路层

在数据链路上传输的数据包，又称为帧（数据链路层是以帧为单位传输和处理的）

数据链路层的协议有很多种，但有三个基本问题是共同的。这三个基本问题是：封装成帧、透明传输和差错检测

  • 封装成帧

加帧头加帧尾，适当配合物理层做定位

  • 透明传输

    保证帧信息解析正确

    可能出现的错误情况

    • 帧具有SOH帧开始符和EOT帧结束符，当数据部分包含了与SOH、EOT相同的字节流，就会导致解析错误，可行的解决方式就是字节填充，在发送帧的时候对每一个数据内部与SOH、EOT或是ESC转义字符相同的在前面追加一个转义字符ESC，这样当接收方解析时，只需要看见一个转义字符与SOH、EOT和ESC组合时，删除一个转义字符即可完成正确解析，也能正确确认一个完整的帧。这样的操作都是在数据链路层上进行，而不需要给网络层带来任何负担

零比特填充法

**MTU(Maximum Transfer Unit)**最大传送单元，规定了帧数据部分的长度上限，在上限以内帧越大越好，一般默认1518B

  • 差错检测

    误码率(Bit Error Rate) 出差的比特占总传输比特的比率

    FCS(Frame Check Sequence)帧校验序列发出方由一些差错校验算法如CRC循环冗余校验计算得出，接收方在收到后计算出一个FCS与接收到的数据帧中的FCS进行比较，判断FCS不能进行纠正，只能进行检错重传-可靠传输服务或者是丢弃差错帧-不可靠传输服务

以太网

以太网是一种计算机局域网技术，IEEE 802.3标准制定了以太网的技术标准，以太网是一种有线系统， 最初使用同轴电缆进行数据传输，后来发展使用双绞线或光纤

每个网络适配器都有一个全球唯一的MAC地址，而交换机和路由器往往拥有更多的网络接口，每个接口都配有一个MAC地址，所以一个路由器或者一个交换机有很多个MAC地址，同一台设备也可以安装多个网卡，配备多个MAC地址，所以MAC地址不能标识主机，只能标识一个网络接口

bit位流从右看向左

MAC帧

进行单播，广播时，目的地址和源地址设定很容易，广播目的地址设置为FF-FF-FF-FF即可，当进行多播时，会发送多播帧，目的地址的MAC值的第一个字节应该为奇数，接收方收到该地址会对自己的多播组列表进行查询，如果多播帧在多播组列表内，则接受该信息

随机MAC地址 目前大多数移动设备都采取了随机MAC地址

交换机

主机A -> 交换机1 ->交换机2 -> 主机B

工作过程：各主机通过交换机的端口接入交换机，主机A在网络层获得待联系的主机B的MAC地址，通过交换机与主机B进行沟通，通过交换机 **存储转发(交换机进行核验在发送)/直接转发(交换机不进行核验)**的方式给主机B发送消息。

  • 问题1 交换机怎么知道主机B在哪个端口

    通过帧交换表和自学习算法，A -> 交换机1，交换机1将A的MAC地址和A的端口记录到帧交换表，交换机查找主机B的记录

  1. 找到了，直接转发到主机B

  2. 找不到，进行泛洪，每一台在交换机中的主机将收到的信息与MAC帧目的地址进行匹配，不对就扔掉，交换机1的消息也传到了交换机2，交换机2先记录A的MAC地址和A在交换机1传入到交换机2的端口，并在交换机2上进行泛洪。

  • 问题2 如果主机从交换机中拔出或两个主机交换，帧交换表怎么更新

    帧交换表存在有效时间，会在有效时间结束后自动删除

稳定性情况：如果 交换机1 -> 交换机2 -> 交换机3或更复杂情况，如果各交换机至今连接唯一没有冗余，那么如果其中一条线故障，则就不能通信，不够稳定

  • 问题1 怎么提高稳定性

    添加冗余线路，比如3个交换机之间两两连线

  • 问题2 如果连线存在冗余线路，出现广播风暴怎么办

    使用STP(Spanning Tree Protocol)生成树算法，构建一条唯一路径，阻塞不必要的路径，如果某条线路出现故障，重新构建路径即可，图唯一路径(最小生成树)

广播情况： 如果交换机结构复杂，根据自学习算法，越大的规模意味着越大的广播域，会占用很多不必要的网络资源。

  • 问题1 如何缩小广播域

    VLAN技术会分割广播域，可以通过交换机进行配置，将各个交换机连接的主机分配进不同的VLAN里，使得同一vlan下的主机可以互相通信，不同VLAN下的主机不能相互通信，或者路由器不同端口会自动分割广播域，但相比交换机更昂贵

*问题2 配置VLAN

1. system-view
2. 创建vlan域
3. 配置端口状态，连接主机的端口设置为access,交换机和交换机之间端口设置为trunk ，即允许多个vlan通多，并设置允许不同vlan通过   
4. 将连接主机端口设置vlan

网络层

路由器分割网络，相同网络号可以互相通信，不同网络之间通信通过路由器，路由器通过路由表进行选择下一跳

IP地址

IPv4的IP地址由网络号和主机号组成，可分为A、B、C、D、E五类

1. A类(网络号8位，第一位恒为0)

00000001|[] - 01111111|[] 1_.0.0.1 - 126_.255.255.254，主机号全0和全1特殊用途不指派，网络号从1到126，最小网络号0保留不指派，最大网络号127留作回环地址

2. B类(网络号16位，前两位恒为10)

1000000000000000|[] - 1011111111111111|[] 128.0_.0.1 - 191.255_.255.254 _符号后为主机号

3. C类(网络号24位，前三位恒为110)

110000000000000000000000|[] - 110111111111111111111111|[] 192.0.0_.1 - 223.255.255.254 _符号后位主机号

子网掩码，借用某IP主机号进一步划分一个网络号

IP协议

用户数据在网络层使用IP协议进行封装，交付给数据链路层，IP协议提供不可靠无连接的数据报传输服务，使用IP协议封装的数据成为IP数据报


第二章ppt p50

IP数据报的发送主机有路由表，查不到默认发给网关，网络号通过ip地址和子网掩码按位与获得，相同网络号直接交付，不同网络号间接交付

路由表

计算机 -> Internet : 首先将目的IP与子网掩码进行按位与运算得到网络号，然后将该网络号与自己的路由表进行匹配，如果匹配到则将数据转发到指定的下一跳，如果没有匹配到，则转发给目标地址为0.0.0.0所对应的下一跳，即默认路由

  • 如果我们发现主机不能够上网或不能ping通外网时，假如检查了IP地址、DNS等设置没有问题依然不能解决，我们可以查看系统的路由表中是否没有添加默认路由

  • 每台主机都有自己的路由表

路由分为动态路由(协议:RIP、OSPF、IS-IS、BGP、IGRP/EIGRP)或静态路由(由网络管理员手动配置的路由)

ARP高速缓存表(获取MAC地址)

ARP协议用于网络层，仅针对同网络下的主机，为主机生成默认时限2分钟的ARP表。

ARP协议通过MAC封装并将目的地址设置为FF-FF-FF-FF来向全部同网络下的主机进行广播，发送ARP请求，请求中包括自己的IP、MAC地址和FF-FF-FF-FF以及目的IP等信息，所有同网络下的主机接收到ARP请求后，先解析目的IP与自己的IP进行按位与运算，如果一致，则将请求方的信息记录在自己的ARP高速缓存表中，并回应一个ARP响应，响应中包括自己的IP和MAC地址，传回到请求方，而目的IP与自己IP不同的主机则将包丢弃。

由于ARP协议没有安全性检验，故可能会遭到攻击机的欺骗
    
1. 主机欺骗：主机A在没有发送ARP请求时，攻击机通过不断向A发送ARP响应，将主机A本来与其他主机通信的MAC修改为自己的MAC地址，IP地址不变，由于ARP协议没有安全性检验，且如果ARP为动态调整，则会直接修改自己的ARP高速缓存表，这样在A向原定主机发送消息通过交换机或路由器时，在数据链路层解析目的MAC时，就会错误解析并将数据发送给攻击机，以达到主机欺骗

2. 网关欺骗：攻击机向路由器网关发送主机A的IP和自己的MAC地址作为ARP响应，修改路由器关于A的路由表，这样在公网给主机A发送数据时，路由器就会错误的把信息发送给攻击机，以达到网关欺骗

TCP


三次握手(核心)


要能够精准描述出来

  • 问题1 为什么不能进行二次握手

如果进行二次握手，那么只要服务端接收到请求就会进入连接已建立，如果该请求是因为网络原因滞后传过来的，但客户端与服务器端已经完成了一次传输，客户端处于关闭状态，服务器达成连接已建立状态向客户发送请求的确认消息，但是因为客户端关闭了，发送不到，那么服务器会一直重传，占用服务器的资源。也就是不能轻易进入连接已建立状态，要有个缓冲

四次挥手(核心)

双方都可以释放连接

  • 问题1 为什么TCP客户进程还需要经过2MSL后才能进入关闭状态

客户关闭就不能打开了，而服务器如果最后一跳报文没有正确传给服务器，那么服务器由于超时重传机制会一直再发，而TCP没有等待直接关闭，那么服务器没法停止发送确认

TCP可靠传输的实现





UDP

单播应用: DNS域名解析 游览器缓存 -> 操作系统缓存 -> 路由缓存 -> ISP DNS服务器 -> 根服务器

多播应用: 网咯视频会议、教学、视频监控等 只需要发一份信息就可以完成通信，不需要针对每一台主机单独进行，节约带宽

广播应用: ARP缓存表

UDP编程，包的大小实际只有1k多，虽然可以达到64k，因为如果分片，有一个传输失败就都要丢弃，还不如直接设置为1k的包

TCP和UDP的区别

TCP只能一对一，UDP可以一对多、多对一和多对多

TCP: 长连接，用于操作频繁，点对点通信，直到某一方主动关闭才结束，短连接，用于操作不频繁的点对点通讯，数据完成就断开
